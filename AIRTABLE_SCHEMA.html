Table 2

Projects

Blueprint source: project-based revenue + retainer revenue tracking. One record = one client engagement. Tracks active projects and recurring revenue simultaneously.

FieldTypeDescription / Formula
project_name ⭐Primary textClient + project type. E.g. "Wanjiru Wholesale — Debtor Tracker"
client_typeSingle selectOptions: Wholesale Distributor, Property Agency, Transport SACCO, Logistics Company, Other
client_nameSingle line textIndividual or business name
client_emailEmailPrimary contact email
tierSingle selectPortfolio (free), Starter (KES 3K–5K), Standard (KES 8K–15K), Premium (KES 15K–25K)
price_kesCurrency (KES)One-time project fee agreed
deposit_receivedCheckbox50% deposit paid. Do not start work until checked.
balance_dueFormula
IF({deposit_received}, {price_kes}/2, {price_kes})
statusSingle selectScoping, Building, Review, Delivered, Retainer Active, Complete, On Hold
start_dateDateDate work began
delivery_dateDatePromised delivery date
days_to_deliveryFormula
DATETIME_DIFF({delivery_date}, TODAY(), 'days')
Shows negative if overdue.
retainer_activeCheckboxIs this client on monthly retainer?
retainer_amount_kesCurrency (KES)Monthly retainer fee (2,500–5,000)
outcome_metricLong textThe specific result delivered. E.g. "3-day reconciliation → automated daily report." Used for case study generation.
testimonial_requestedCheckboxHave you asked for a testimonial? Check on delivery day.
case_studyLink to Case StudiesLink to the case study derived from this project.
notesLong textInternal notes, scope notes, client communication log.

Views

Grid
Active Projects
Everything in progress right now.
Filter: status IN (Scoping, Building, Review)
Grid
Retainer Clients
Recurring revenue. Call each one monthly.
Filter: retainer_active = TRUE
Grid
Overdue Delivery
Projects past their delivery date.
Filter: days_to_delivery < 0 AND status ≠ Delivered
Grid
Awaiting Testimonial
Delivered but testimonial not yet requested.
Filter: status = Delivered AND testimonial_requested = FALSE

Table 3

Case Studies

Blueprint source: case studies convert mid-funnel buyers better than testimonials. Each record populates one portfolio case study page. The portfolio reads this table via Airtable API.

FieldTypeDescription
title ⭐Primary textFormat: "[Client Type] — From [Before State] to [After State]"
slugSingle line textURL-safe version of title. E.g. "wholesale-distributor-debtor-tracker". Used in portfolio URL.
client_typeSingle selectWholesale Distributor, Property Agency, Transport SACCO, Logistics, Other
problem_stateLong textThe before state. How was the business operating before your system? Be specific.
system_builtLong textWhat you built. Tables, automations, views. Technical but readable.
outcome_metricSingle line textThe one-line result. E.g. "KES 200K in outstanding debt made visible on day 1."
pipeline_stagesMulti selectStructure, Automate, Illuminate (which stages this project involved)
loom_urlURLLoom walkthrough video. Record a 90-second demo of the live system.
hero_image_urlURLScreenshot of the Airtable system. Upload to Cloudinary, paste URL here.
tools_usedMulti selectAirtable, Google Sheets, n8n, Zapier, SQL, Python, Supabase
published_dateDateWhen published to portfolio. Portfolio sorts by this descending.
publishedCheckboxOnly checked records appear on portfolio. Keep unchecked while drafting.
linked_projectLink to ProjectsThe project this case study comes from.
Every case study must have a real outcome_metric. "Helped the client manage their data better" is not an outcome metric. "KES 180,000 in untracked customer debt made visible on day one" is an outcome metric.

Leads

Blueprint source: LinkedIn comment → DM → discovery call → project acquisition funnel. Every prospect from every source lands here. This table tells you which content and which source is generating revenue.

FieldTypeDescription
name ⭐Primary textFull name of prospect
emailEmailContact email
businessSingle line textBusiness name and location. E.g. "Kariuki Wholesale, Gikomba"
problem_descriptionLong textWhat they said their problem is. Exact words if possible.
sourceSingle selectLinkedIn DM, LinkedIn Comment, Portfolio Website, WhatsApp, Referral, Upwork, Walk-in, Other
statusSingle selectNew → Contacted → Discovery Call Booked → Discovery Call Done → Proposal Sent → Won → Lost → No Response
submitted_atDate + timeWhen lead came in. Auto-set by website form via API.
discovery_call_dateDateScheduled discovery call date
proposal_amount_kesCurrency (KES)The amount you proposed
proposal_sent_dateDateWhen proposal was sent
follow_up_dateDateWhen to follow up next. Set manually after each interaction.
days_since_contactFormula
DATETIME_DIFF(TODAY(), {submitted_at}, 'days')
lost_reasonSingle selectPrice, Timing, No Response, Chose Competitor, Not a Fit. Fill when status = Lost.
converted_projectLink to ProjectsLink to project record when lead converts. Shows full funnel.
notesLong textNotes from each interaction. Date-stamp each: "12 Jun — called, needs a week to decide"

Views

Grid
Action Required Today
Leads with follow_up_date = today. These people need a message from you right now.
Filter: follow_up_date = today AND status ≠ Won/Lost
Grid
Hot Pipeline
Discovery call done or proposal sent. Closest to revenue.
Filter: status IN (Discovery Call Done, Proposal Sent)
Grid
Gone Cold (14+ days)
No contact in 14 days. Either re-engage or mark Lost.
Filter: days_since_contact > 14 AND status = Contacted
Grid
By Source
Grouped by source. Shows which channel generates actual revenue.
Group by: source · Sort: status

Content Planner

Blueprint source: 6 content pillars, 4 posts/week, observation-first method. Tracks posts from idea to published, then links performance back to which content sources produce leads.

FieldTypeDescription
post_title ⭐Primary textWorking title of the post. Start with the observation, not the lesson.
pillarSingle select1-Proof (system screenshots), 2-Observation (business reality), 3-Process (how Structure/Automate/Illuminate works), 4-Insight (patterns from client work), 5-Question (engagement), 6-Failure (what went wrong and why)
observation_notesLong textThe raw observation. What did you notice in a real business? Write it here before you write the post.
post_bodyLong textFull post copy. Write here before scheduling.
statusSingle selectIdea → Draft → Ready → Scheduled → Published
scheduled_dateDateWhen to post. Blueprint: Tuesday + Friday = highest engagement for this audience.
published_dateDateActual publish date
linkedin_urlURLLink to the live LinkedIn post once published.
impressionsNumberLinkedIn impressions (check 48 hours after posting)
commentsNumberComment count. Comments > impressions for lead generation.
leads_generatedRollup from LeadsCount of leads whose source = this post. Tells you exactly which content converts.
Grid
This Week
Posts scheduled this week. Check Monday morning.
Filter: scheduled_date = this week
Grid
Ready to Post
Written and approved. Schedule these.
Filter: status = Ready
Grid
Top Performers
Sort by impressions then by comments to find what resonates.
Sort: comments DESC
Grid
Ideas Bank
Observations noted but not yet drafted.
Filter: status = Idea
Table 6

Testimonials

Blueprint source: specific operational metric + named client type = mid-funnel conversion trigger. Only real testimonials with permission. Never fabricate.

FieldTypeDescription
client_name ⭐Primary textFull name. First name only acceptable if client prefers.
roleSingle line textJob title or role. E.g. "Director", "Owner", "Operations Manager"
companySingle line textBusiness name. If client prefers anonymity, use "[Wholesale Distributor, Nairobi]"
quoteLong textExact words. Do not edit for grammar — authentic beats polished.
permission_grantedCheckboxMUST be checked before this appears on portfolio. Screenshot or record the permission.
contains_metricCheckboxDoes the quote contain a specific number or outcome? Metric-based testimonials convert 3x better.
date_receivedDateWhen you received the testimonial.
linked_projectLink to ProjectsThe project this testimonial relates to.
permission_evidenceAttachmentScreenshot of WhatsApp/email where client approved use. Legal protection.
Blueprint rule: one testimonial that says "I can now see exactly who owes me KES 200,000 — before this I had no idea" is worth more than ten that say "Waren did a great job." When requesting, ask specifically: what changed for your business after the system was built?
